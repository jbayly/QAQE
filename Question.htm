Business Requirement:
 
Your task is to pay for a particular invoice from a list of invoices.
 
Select an invoice you need to pay from the list of given invoices. Tax is levied on every invoice payment and the tax imposed is 10% of the payable amount. Once the payment is complete,update the balance amount and increase payment attempt count by one. Validate the credit card and if all the details are valid, display the total payable amount for the invoice along with the balance amount pending for payment. Handle the exceptions if any entered credit card details do not comply with the validation rules. Along with this, you also need to streamline to entity / java class structure to align with object oriented programming principles like inheritance ,abstract class, exception and jdbc.
 
Requirement 1:
 

The Invoice class has following private member variables/attributes.

Datatype Name Description
Integer id Unique Database identifier
String customerName Name of the customer
Integer paymentAttempts No. of attempts, the customer attempt to pay the invoice amount.
Double totalAmount Total invoice amount
Double balance Pending invoice amount
String status Status of the invoice

The abstract class Payment with the following protected attributes

Datatype Name Description
String accountHolderName The name of the bank account holder.
Double amount Amount going to pay.

The class CreditCardPayment which extends Payment with the following private attributes

Datatype Name Description
String cardNumber Unique identity of card
String cvv Cvv number of card
String cardName Name of the card

For the above classes,appropriate  constructor,getters, setters and default constructors are already given as part of the template code.

The InvalidPaymentException which extends Exception and act as custom exception.
Include a constructor with single argument. Argument to this constructor is message.

 Include an abstract method calculateTotalAmount() in the Payment class
  •     public abstract Double calculateTotalAmount()
The class CreditCardPayment should extend the abstract class Payment and this class has to override all the abstract methods in the Payment class.
 


Requirement 2:

The class CreditCardPayment has the following methods
No Class Name Method Name Input Parameters Return type
1 CreditCardPayment Define calculateTotalAmount()
Validate the card details. If the details are invalid then throw InvalidPaymentException with the message “Invalid Card Details” else
returns the total payable amount. Total payable amount is the sum of amount and 10% of the amount.

Validation Rules:
1.    The card number should be 16 digits and should contain only numbers.
2.    The cvv should be only 3 digits
Note:
If any one or both the validation rule fails, then throw an exception
 		 - Double

Input and Output Format :
 
Input:
First input is an integer that corresponds to the invoice id.
The following inputs corresponds to the payment and credit card details.
 
Output:
Invoice details should be of the format:
System.out.format("%-5s %-15s %-20s %-15s %-10s %s\n","ID","Customer Name","Payment Attempts","Total Amount","Balance","Status")
Display the total payable amount and the balance invoice amount still pending for payment in two decimal places.
If the payment details are invalid, display the exception message.
